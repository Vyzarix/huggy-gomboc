## 2026-06-13

After several months of planning, I finally created my GitHub profile and a repository to document the development of my Huggy Gomboc project.

## 2026-06-23

I planned to order the first components for Huggy Gomboc V0.1.

Main components:

- LM2596 buck converter
- WS2812B 12 LED ring
- KY-006 passive buzzer
- TTP223 capacitive touch sensor
- MG90S 180° servo

The main goal of V0.1 is to build the first simple physical reaction system:

touch → LED heartbeat → buzzer → small servo movement

The next step is to study my existing ESP8266 robot kit and understand its factory example codes, especially motor control, sensor reading and basic movement.

## 2026-06-24 – Huggy AI Direction Started

The first focused direction for Huggy AI started on June 24, 2026.

The goal was to create more than a simple chatbot: Huggy AI should become the emotional and conversational layer of the Huggy Gomboc project. The first idea was to connect personality, memory, emotional reflection, and later physical robot reactions into one system.

Core idea:

`emotion → memory → conversation → internal state → future physical reaction`

At this stage, Huggy AI was still only a concept, but the direction became clear: a small companion AI layer that can remember, listen, respond gently, and later connect to a physical body.

## 2026-06-25 – First Working Terminal Prototype

Huggy AI V0.1 had its first working terminal launch in the early morning of June 25, 2026.

The first prototype could:

* run from the terminal;
* load a personality file;
* use a memory file;
* use a journal file;
* save conversations into timestamped files;
* respond through an OpenAI API call.

This was the first real software milestone of Huggy AI.

The project moved from an idea into a working local prototype.
