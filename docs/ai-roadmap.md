# Huggy AI Roadmap

## V0.1 – Terminal Chat Prototype

Huggy AI can start in the terminal and respond to the user.

Main goals:

* run locally from a Python script;
* accept text input from the terminal;
* send the conversation to an AI model;
* print Huggy's response back to the terminal.

## V0.2 – Basic Memory and File Structure

Huggy AI has a basic personality and memory structure.

Main goals:

* use `personality.md` for Huggy's character and response style;
* use `memory.json` for stable long-term memories;
* use `journal.md` for emotional and project-related notes;
* save full conversations into a `conversations/` folder.

## V0.3 – Refined Personality and Emotional Modes

Huggy AI becomes more personal and emotionally aware.

Main goals:

* refine Huggy's speaking style;
* add an evening love / longing / emotional reflection mode;
* reduce overly technical or task-like responses during emotional conversations;
* support selected private long-term memories stored locally;
* improve Huggy's ability to listen, ask gentle questions, and stay present.

## V0.4 – Improved Memory System and Conversation Recall

Huggy AI gains a better long-term memory system.

Main goals:

* read more than only the end of `journal.md`;
* search older memories when needed;
* separate personal memories, project memories, and emotional themes;
* recall older conversations more intelligently;
* support different modes, such as emotional mode, project mode, technical mode, and reflection mode.

## V0.5 – Voice Output / Text-to-Speech

Huggy starts to speak out loud.

Main goals:

* convert Huggy's text responses into speech;
* play responses through a speaker;
* experiment with different voice styles;
* keep the voice soft, calm, and fitting for Huggy's personality.

## V0.6 – Microphone Input / Speech Recognition

Huggy starts to listen.

Main goals:

* record audio from a microphone;
* convert speech into text;
* send the recognized text into Huggy AI;
* allow basic spoken conversations.

## V0.7 – Raspberry Pi Integration

Huggy AI starts moving toward a physical robot platform.

Main goals:

* run parts of the system on a Raspberry Pi;
* use the Raspberry Pi as the main logic and communication layer;
* connect the AI layer to sensors, sound, and physical reactions;
* keep heavier AI models either online or on a stronger computer if needed.

## V1.0 – Physical Huggy Prototype

Huggy becomes a basic physical companion robot.

Main goals:

* LED reactions;
* servo movement;
* button or touch input;
* basic sound output;
* simple voice interaction;
* AI mood mapped to physical behavior;
* basic conversation with visible physical presence.
